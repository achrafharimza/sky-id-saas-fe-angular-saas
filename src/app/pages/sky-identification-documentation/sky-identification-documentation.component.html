La SDK Web permettant de communiquer en mode streaming avec le système d'identification faciale Sky Deep Face Match en comparant plusieurs photos apposée sur la pièce d'identité prises via l'OCR Streaming et plusieurs photos (selfie) prises via la vidéo streaming afin de s'assurer l'opération KYC en s'assurant que le détenteur de la pièce d'identité est le même que la personne qui est devant la caméra en train de faire KYC, il y a plusieurs niveaux :

  • Level 1 : Matching document VS selfie
  • Level 2 : Matching document VS video streaming
  • Level 3 : Matching video ID Card Streaming VS video Faciale streaming

The SkyID WEB SDK enables your app to analyze, liveness verification and face matching using biometrics based solutions.

Web SDK files :

  • skyid_fa_sdk-min.js : javascript module of sky face analysis, contain two APIs :
    • skyid_fa_setup_sdk : enable loading and initializing sdk components(webcam, engines, elements positioning).
  • skyid_fa_start_face_scan : run face detection and liveness check.
  • skyid_fa_sdk_bundles.js : javascript file for editing SDK messages or html elements IDs.
dependance : jquery, tfjs.

Plus, HTML components to push in your web page (see project demo).

Web SDK APIs implementations :

  1. Setup SDKs, you need to provide the token and application id : 
      const skyid_fa_sdk_token = "***************************************************";
  const skyid_fa_sdk_appId = "aa.aa.aaa";
  skyid_fa_setup_sdk(skyid_fa_sdk_token, skyid_fa_sdk_appId);
  2. Running face scan, the API skyid_fa_start_face_scan prend en parametre two function callbacks:
    • skyid_fa_end_api_check_callback : triggered when the SDK returns Liveness check from the skyid API. two data parameters is returned :
      • skyid_fa_selected_frame : a canvas of the user selfie.
      • skyid_fa_response_dict : contain the liveness check response, is a dictionary of :
        1. status_code : the api status code, 000:success otherwise is an error described in status label.
        2. status_label : a text describing the API response.
        3. class : contain reel : live face, spoof: attack, None: api error.
        4. score : the liveness checks confidence.
    • skyid_fa_end_with_spoof_callback : triggered when the SDK does not detect a live face in the browser.
